在 ROS 2 的零拷贝实现中，应用层、DDS 和 Linux 系统调用各自扮演着不同的角色，它们协同工作，共同完成高效的数据传输。

# 各层级在零拷贝中的角色

---

## 1. 应用层 (ROS 2 节点)

应用层是用户直接编程的层面，它使用 ROS 2 的 API 来发送和接收消息。在这里，零拷贝的角色主要是**内存管理和数据填充**。

- **申请内存**：通过调用像 `publisher->borrow_loaned_message()` 这样的 API，应用层**向 DDS 借用一块内存**。这块内存是 DDS 已经分配好的，而不是应用层自己分配。
- **填充数据**：应用层直接在这块借来的内存上写入数据（例如，图像像素或点云数据），避免了在自己的缓冲区和 DDS 缓冲区之间进行数据拷贝。

应用层只负责使用这些 API，它并不关心底层的具体实现，比如内存是如何分配的，数据又是如何发送的。

---

## 2. DDS (Data Distribution Service)

DDS 是 ROS 2 的通信骨干，它在应用层和 Linux 系统调用之间扮演着**关键的中间件角色**。DDS 不仅负责消息的发布和订阅，更重要的是，它管理着零拷贝的**核心逻辑和内存池**。

- **内存池管理**：DDS 维护一个或多个**内存池**（memory pool）。当应用层请求“贷款”消息时，DDS 会从这些内存池中分配一块内存的引用给应用层。
- **零拷贝实现**：当应用层填充完数据并调用 `publish` 时，DDS 会直接使用这块内存，调用底层的网络 I/O 函数（如 `sendfile` 或 `splice`），将数据从内存池直接发送到网络缓冲区。
- **消息序列化和封装**：DDS 还会对数据进行序列化，并添加必要的协议头信息，但这些操作通常是在内存中原地完成的，不会引发额外的数据拷贝。

**为什么要有 DDS 的事情？**

DDS 的存在至关重要，因为它**将底层复杂的零拷贝实现细节封装起来**，并提供了一个跨平台的、可靠的通信抽象。ROS 2 并没有直接操作 Linux 系统调用，而是依赖 DDS 来完成这些任务。DDS 确保了：

1. **跨平台兼容性**：ROS 2 可以在 Linux、macOS 和 Windows 上运行，而每个操作系统的底层零拷贝技术都不同。DDS 负责适配这些差异。
2. **协议和 QoS 管理**：DDS 不仅仅是发送数据，它还提供了丰富的质量服务（QoS）设置，如可靠性、延迟容忍度、持久性等，这些都是零拷贝功能无法单独提供的。
3. **多播和发现机制**：DDS 自动处理节点发现和多播通信，这使得零拷贝的端到端通信更加高效和简单。

---

## 3. Linux 系统调用

Linux 系统调用是**实际执行零拷贝操作的底层机制**，它们是操作系统提供的功能接口。DDS 依赖这些系统调用来完成最终的数据传输。

- `**splice()**` 或 `**sendfile()**`：DDS 会使用这些系统调用来**直接在内核空间完成数据传输**。例如，它可以将 DDS 内存池中的数据（位于内核空间）直接移动到网络套接字的缓冲区，完全跳过用户空间的数据拷贝。
- `**mmap()**`：DDS 也可能使用 `mmap()` 将文件或设备内存直接映射到其内存池，从而实现零拷贝的文件 I/O。

## 总结

三者的关系可以比喻为一个三层楼的建筑：

- **顶层（应用层）**：你只知道按下电梯按钮（`borrow_loaned_message`）就能到达目的地。
- **中间层（DDS）**：电梯本身，它负责调度、管理乘客（数据）和控制升降。它知道如何利用楼宇的底层结构（Linux 系统调用）来高效地运行。
- **底层（Linux 系统调用）**：电梯井道和机械结构，是实际执行运动的物理设施。电梯（DDS）是使用这些设施的人。

因此，ROS 2 的零拷贝是一个**分层协作的成果**。DDS 作为关键的中间层，不仅提供了零拷贝的能力，更将其集成到 ROS 2 的通信框架中，使得开发者可以在不深入了解底层技术细节的情况下，享受到零拷贝带来的性能优势。

因此，零拷贝技术的目的不是消除两次 DMA 拷贝，而是**消除 CPU 的参与拷贝，以及由此产生的额外数据拷贝**。

- **传统 I/O**：数据从硬盘到内存（DMA），再到 CPU 可见的缓冲区，再到 Socket 缓冲区，最后再到网卡（DMA）。这里面存在多次由 CPU 参与的、不必要的**内存到内存**的拷贝。
- **零拷贝技术（如** `**sendfile()**`**）**：数据从硬盘到内存（DMA），然后直接从内存到网卡（DMA）。这个过程中，数据只在内核缓冲区中停留了一次，**避免了从内核缓冲区到用户缓冲区，再到 Socket 缓冲区的多次 CPU 拷贝**。

所以，“零拷贝”不是指完全没有数据拷贝，而是指**减少甚至消除那些由 CPU 完成的、无谓的数据拷贝**。两次 DMA 拷贝是数据从一个硬件到另一个硬件的**物理性**搬运，是无法省略的。
