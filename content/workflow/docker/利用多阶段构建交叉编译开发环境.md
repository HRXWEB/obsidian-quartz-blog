# docker 配置

1. 安装docker: [https://docs.docker.com/engine/install/](https://docs.docker.com/engine/install/)
2. 启用 buildx 功能（这是 docker 的实验性功能，帮助我们进行[多架构构建](https://docs.docker.com/build/building/multi-platform/)）
    1. 检查 docker 版本至少是 19.03 及以上： `docker --version`
    2. 启用 buildx
        
        ```Shell
        docker buildx create \
          --name container-builder \
          --driver docker-container \
          --driver-opt network=host \
          --driver-opt env.http_proxy=http://127.0.0.1:7890 \
          --driver-opt env.https_proxy=http://127.0.0.1:7890 \
          --bootstrap --use
        ```
        
3. 设置平台支持。为了支持 ARM64 架构的构建，需要安装 QEMU 模拟器，这样可以让 docker 在 x64 机器上模拟 ARM 架构：
    
    ```Shell
    # 安装 qemu
    sudo apt-get install qemu qemu-user-static
    # 将 QEMU 注册到 docker，使其支持 ARM64 架构
    docker run --rm --privileged multiarch/qemu-user-static --reset -p yes
    ```
    

# 编写 dockerfile

下面有一个完整的例子，考虑模块化的问题

[cc_multi_stage_docker_demo.tar.gz](https://cdn.jsdelivr.net/gh/hrxweb/obsidian-assets@main/assets/cc_multi_stage_docker_demo.tar.gz)

其中 dockerfile 的内容为：

```Docker
# 阶段1：构建 ARM64 rootfs
FROM --platform=linux/amd64 ubuntu:20.04 AS build_rootfs

# 配置清华源
RUN sed -i 's/archive.ubuntu.com/mirrors.tuna.tsinghua.edu.cn/g' /etc/apt/sources.list && \
    sed -i 's/security.ubuntu.com/mirrors.tuna.tsinghua.edu.cn/g' /etc/apt/sources.list

# 安装构建 rootfs 所需要的工具
RUN apt update && apt install -y debootstrap

# 创建 rootfs 目录，并使用 debootstrap 创建 ARM64 的最小 Ubuntu 系统
RUN mkdir /rootfs && debootstrap --arch=arm64 focal /rootfs https://mirrors.tuna.tsinghua.edu.cn/ubuntu-ports/


# 阶段2：进入 rootfs 并安装额外的包
FROM --platform=linux/amd64 ubuntu:20.04 AS install_packages

# 将第一个阶段的 rootfs 复制到当前构建阶段
COPY --from=build_rootfs /rootfs /rootfs

# 使用 chroot 进入 rootfs 安装软件包
COPY target_scripts/target_scripts.sh /rootfs/target_scripts.sh
COPY target_scripts/fish_install.yaml /rootfs/fish_install.yaml
RUN chmod +x /rootfs/target_scripts.sh
RUN chroot /rootfs /bin/bash -c "bash target_scripts.sh"


# 阶段3：执行后续镜像配置和操作
FROM --platform=linux/amd64 ubuntu:20.04 AS final
WORKDIR /workspace

# 将构建好的 rootfs 复制到最终镜像中
COPY --from=install_packages /rootfs /rootfs

COPY host_scripts/toolchain.cmake toolchain.cmake
COPY host_scripts/host_scripts.sh host_scripts.sh
COPY host_scripts/fish_install.yaml fish_install.yaml
RUN chmod +x host_scripts.sh
RUN bash host_scripts.sh
COPY scripts/fix_cmake_files.py fix_cmake_files.py
RUN python3 fix_cmake_files.py /rootfs
COPY scripts/fix_rootfs_softlink.py fix_rootfs_softlink.py
RUN python3 fix_rootfs_softlink.py /rootfs

# Config ssh port 2345
RUN apt update && apt install -y openssh-server
RUN sed -i 's/\#Port 22/Port 2345/' /etc/ssh/sshd_config && \
    sed -i 's/\#PermitRootLogin prohibit-password/PermitRootLogin yes/' /etc/ssh/sshd_config \
    && echo "root:root" | chpasswd

# Expose 2345 port
EXPOSE 2345

# Start ssh service and enter bash
CMD service ssh start && bash
```

## 解释

整个过程分为三个阶段：

1. 第一个阶段先从 apt 源中，利用 debootstrap 工具制作基础的 rootfs。 apt 源有一个值得注意的事情： [[amd64 平台和 arm64 平台的 apt 源]]
2. 第二个阶段利用 chroot 在 rootfs 文件系统中，执行 `target_scripts.sh` 安装需要的依赖
3. 第三个阶段执行 `host_scripts.sh` 在 amd64 平台安装必要的依赖

因此：

> 只需要替换 `target_scripts.sh` 文件和 `host_scripts.sh` 文件就可以开发适用于别的项目的交叉编译环境。做到了良好的模块化。

## 两个有用的脚本

- `fix_cmake_files.py` ： 为了交叉编译，修复依赖包的 `.cmake` 文件中的绝对路径问题，在其其面加上 `${CMAKE_SYSROOT}` 前缀，这样就避免了把 host 平台的文件混入了编译过程。
- `fix_rootfs_softlink.py` ： 有一些软链接，比如动态库的软链接，指向的路径是绝对路径，由于 rootfs 的存在可能导致绝对路径失效，或者找到 host 平台的 incompatible format 文件。这个脚本会将其改为相对路径，指向正确的文件。
