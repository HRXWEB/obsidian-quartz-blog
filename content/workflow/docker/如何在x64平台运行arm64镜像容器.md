1. 安装 qemu 和 docker 多架构支持
    
    ```Shell
    sudo apt update
    sudo apt install -y qemu qemu-user-static binfmt-support
    ```
    
2. 安装docker
3. 启用多平台支持 buildx
    
    ```Shell
    docker buildx create \
      --name container-builder \
      --driver docker-container \
      --driver-opt network=host \
      --driver-opt env.http_proxy=http://127.0.0.1:7890 \
      --driver-opt env.https_proxy=http://127.0.0.1:7890 \
      --bootstrap --use
      
    # 如何再次启动:
    docker buildx inspect container-builder --bootstrap
    # 通过如下命令确定真的在 running
    docker buildx ls
    ```
    
4. 安装 qemu 支持
    
    ```Shell
    docker run --rm --privileged multiarch/qemu-user-static --reset -p yes
    ```
    
5. 验证是否识别 arm64 平台
    
    ```Shell
    docker run --rm --platform linux/arm64 arm64v8/alpine uname -m
    ```
    

---

- 正常运行 arm64 容器
    
    ```Shell
    docker run --rm --platform linux/arm64 arm64v8/alpine uname -m
    ```
    
- 构建 arm64 镜像
    
    ```Shell
    docker buildx build --load --platform linux/arm64 -t my-arm64-image .
    ```
    
    `--load` 参数最后会将 layer export 成 image 到 x64平台，否则 `docker images` 命令看不到这个镜像。需要注意⚠️的是此参数不能和多平台构建一起使用，即 `--platform` 只能指定一个平台。
