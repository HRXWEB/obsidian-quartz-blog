# Clang v.s. GNU GCC

## 标准库

|     | Clang  | GNU GCC   |
| --- | ------ | --------- |
| c   | glibc  | glibc     |
| c++ | libc++ | libstdc++ |

Q：交叉编译工具链都会自带标准库的实现，clang 也有自己实现的标准库，会链接到哪一个呢？

（兼容 libstdc++，默认是 libc++，通过 -stdlib 指定)

# 换源

```bash
cp -a /etc/apt/sources.list /etc/apt/sources.list.bak
sed -i "s@http://.*archive.ubuntu.com@http://mirrors.huaweicloud.com@g" /etc/apt/sources.list
sed -i "s@http://.*security.ubuntu.com@http://mirrors.huaweicloud.com@g" /etc/apt/sources.list
```

宿主机：

1. `apt update && apt install rsync clang libomp-dev -y`

rootfs:

1. `apt update && apt install libomp-dev`

原因是 openmp 链接的库有问题，但没有具体定位到问题，但还是解决了：

```bash
if (OPENMP_FOUND)
    message(STATUS "OpenMP found")
    # 注视下面这一行
    # add_compile_options(${OpenMP_C_FLAGS} ${OpenMP_CXX_FLAGS})
    # 换成下面这两行
    set(CMAKE_C_FLAGS "${CMAKE_C_FLAGS} ${OpenMP_C_FLAGS}")
    set(CMAKE_CXX_FLAGS "${CMAKE_CXX_FLAGS} ${OpenMP_CXX_FLAGS}")
    # 保持其他的不变
    link_libraries(${OpenMP_EXE_LINKER_FLAGS})
endif ()
```
